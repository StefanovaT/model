Table Cm_Party_Contact_Mechanisms

Entity

Entity: General.Contacts.PartyContactMechanisms

Specifies the contact mechanisms, which are attached to the parties. Currently each contact mechanism is attached to strictly one party. Entity: Cm_Party_Contact_Mechanisms

Owner Tables Hierarchy

  • Gen_Parties

Summary

Name Type Description
Contact_Mechanism_Id uniqueidentifier The contact mechanism of the party
Contact_Mechanism_Purpose_Id uniqueidentifier The purpose of this contact mechanism. Unique within the party. Can be used to seek for specific contact mechanisms.
From_Date datetime The first date when the contact mechanism was valid. NULL means unknown date
Id uniqueidentifier PK Unique party contact mechanism Id
Is_Active bit True if the contact mechanism is currently active and can be used to contact the party.
Is_Default bit 1 - when this is the default contact mechanism for this party; 0 - otherwise
Line_Ord int Consecutive number of the contact information. The number is unique within the party.
Non_Solicitation bit If 1 then Don't use the mechanism for solicitation purposes
Notes nvarchar(254)
Party_Id uniqueidentifier The party, having the contact mechanism
Personal_Data_Process_Id uniqueidentifier The personal data process, which is used to process the current data. Null when the data is not personal or when the process is unknown.
Row_Version timestamp
Thru_Date datetime The last date on which the contact mechanism was valid for the party. NULL if the contact mechanism is still valid
