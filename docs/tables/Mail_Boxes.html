Table Mail_Boxes

Entity

Entity: Applications.Mail.Boxes

Represents user mailboxes. Entity: Mail_Boxes

Summary

Name Type Description
Active bit 1 when the mail box is active for mail sinchronization, sending and receiving
Email_Address nvarchar(254) The email address associated with this mail box
Is_Default bit 1 when this is the default mailbox for the user.
Is_Encrypted bit 1 when the mailbox messages are stored in encrypted format
Mail_Box_Id uniqueidentifier PK
Owner_User_Id uniqueidentifier The user, who owns the mailbox
Reset_On_Next_Sync bit Readonly Reset is pending: 1 = next sync should delete and retrieve again all messages; 0 = no pending reset
Row_Version timestamp
Send_Server_Address nvarchar(254) The address of the server to use for sending mail. The same credentials as the sync server are used. When NULL, the mail will be sent using the default server, which is setup at the application server
Sent_Items_Folder_Name nvarchar(254) The name of the SentItems folder. It is left empty, EnterpriseOne won't save the sent mail in any folder (but it is still possible that the actual mail server would nevertheless save the mail in a sent items folder, independently from EnterpriseOne).
Signature_Html nvarchar(max) Html text of the default signature, when creating new emails from this mailbox. When is NULL, a default generic signature is attached
Sync_Password nvarchar(512) The password to supply to the server when retrieving email. NULL when the connection is not setup or the server does not require user name
Sync_Protocol nvarchar(4) Allowed: IMAP, POP3, MEWS, NSNC Synchronization protocol, 'POP3' or 'IMAP'. NULL means that syncrhonization is not setup
Sync_Server_Address nvarchar(254) Incoming and outgoing mail server internet address. NULL when synchronization is not setup
Sync_User_Name nvarchar(64) User name to supply to the server, when retrieving email. NULL when the connection is not setup or the server does not require user name
