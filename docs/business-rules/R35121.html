R35121 ExciseStampOperations - Excise Product Type And Excise Stamp Lot Excise Product Type Cannot Be Different

Code R35121
Entity Finance.Excise.ExciseStampOperations Entity
Name ExciseProductTypeAndExciseStampLotExciseProductTypeCannotBeDifferent
Attribute ExciseProductType, ExciseStampLot
Layer Back-End
Events Releasing +
Priority Normal
Modify NO
Action For each line, where Excise Stamp Lot and Excise Product Type are selected - the rule validates that Excise Product Type is equal to the Excise Product Type, specified in the Excise Stamp Lot definition.
Description When the Excise Stamp Operation document's state is changed to "Released" or higher, the rule ensures that the lines do not contain excise product types that are different from the excise product type of the selected excise stamp lots.
Message The Excise Product Type field in line number '{ExciseStampOperationLine.LineNo}' in document '{DocumentText}' contains an invalid value.

The excise product type '{ExciseStampOperationLine.ExciseProductType}' is different from the one specified in the definition of the excise stamp lot '{ExciseStampOperationLine.ExciseStampLot}'.

Please make sure that the selected еxcise product type matches the excise product type in the excise stamp lot definition and try again.
Version Introduced: v.24
Revocable YES
